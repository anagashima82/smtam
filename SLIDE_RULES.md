# Marpスライド作成ルール

このドキュメントは、Marpを使った講義資料スライド作成時のルールとガイドラインを定めたものです。

---

## 1. ファイル構成

### フロントマター（必須）

```yaml
---
marp: true
theme: custom
footer: "**SAMURAI [講座名]**"
paginate: true

title: "[章番号] [タイトル]"
subtitle: "[サブタイトル]"
author: "SAMURAI"
date: "YYYY-MM-DD"

---
```

---

## 2. スライドの基本構成

### 章の構成順序

1. **タイトルスライド** - 講座全体のタイトル
2. **目次スライド**（複数章ある場合）
3. **章タイトルスライド** - `<!-- _class: title -->`
4. **章概要スライド** - 目標と想定学習時間
5. **本編スライド** - 内容の解説
6. **まとめスライド** - 学習内容の振り返り

### スライドクラス

| クラス | 用途 |
|--------|------|
| `<!-- _class: title -->` | タイトルスライド（中央寄せ、大きい文字） |
| `<!-- _class: title-top -->` | 通常のコンテンツスライド |

---

## 3. 見出しルール

### 見出しレベルの使い分け

| 見出し | 用途 | 例 |
|--------|------|-----|
| `#` | 章タイトル（titleクラスのみ） | `# 1章 Pythonの概要を理解しよう` |
| `##` | スライドタイトル | `## 1章 Pythonとは` |
| `###` | セクション内の小見出し | `### scikit-learn` |

### スライドタイトルの命名規則

```markdown
## [章番号] [トピック名]
```

例：
- `## 1章 Pythonとは`
- `## 1章 特徴❶ 初学者でも比較的簡単に習得できる`
- `## 1章 まとめ`

---

## 4. レイアウトパターン

### 1カラムレイアウト（基本）

```markdown
## [タイトル]

### [最初の文章・重要なポイント]

- ポイント1
- ポイント2
- ポイント3
```

**重要**: タイトルスライド以外のページでは、最初の文章は必ず `###` で始める

### 2カラムレイアウト

```markdown
## [タイトル]

<div class="flex sa">
<div>

### [左カラムタイトル]

- 内容1
- 内容2

</div>
<div>

### [右カラムタイトル]

- 内容1
- 内容2

</div>
</div>
```

### 3カラムレイアウト

```markdown
## [タイトル]

<div class="flex-3col">
<div>

### [タイトル1]

内容

</div>
<div>

### [タイトル2]

内容

</div>
<div>

### [タイトル3]

内容

</div>
</div>
```

## 5. スタイル要素

### 太字の強調（重要）

リスト内やピンク背景ボックス内で太字を正しく表示するには、`<div class="no-bg">` で囲む：

```markdown
<div class="no-bg">

**重要なテキストをここに記述します。**

</div>
```

### 丸数字の使用

番号付きの項目には丸数字を使用：

- ❶ ❷ ❸ ❹ ❺ ❻ ❼ ❽ ❾ ❿

例：
```markdown
### ❶ 習得しやすい
### ❷ 需要が高い
### ❸ ライブラリが豊富
```

### 画像の配置

```markdown
<div class="image-center">

![width:1000px](./images/image-name.png)

</div>
```

- 画像幅は基本 `width:1000px`
- 外部画像URLも使用可能

## 6. コンテンツガイドライン

### 1スライドの情報量

- **箇条書き**: 3〜5項目が目安
- **文章**: 長文は避け、簡潔に
- **画像スライド**: 画像のみ、または画像＋短い説明

### 説明と図解のペア

複雑な概念は「説明スライド」と「図解スライド」をセットで作成：

1. `## 1章 データベースとは` → テキストで説明
2. `## 1章 データベースとは` → 図解（同じタイトルでOK）

### 表（テーブル）の使用

比較や用語説明に使用：

```markdown
| 用語 | 説明 |
|------|------|
| **テーブル** | 1つひとつの表 |
| **カラム** | テーブルにおける縦1列 |
```

## 7. 章の開始と終了

### 章タイトルスライド

```markdown
---

<!-- _class: title -->

# [章番号] [章タイトル]
---
```

### 章概要スライド

```markdown
---
<!-- _class: title-top -->

## [章番号] [章タイトル]

[章の概要を1〜2文で記述]

**本章の目標**
- 目標1
- 目標2

---
```

**注意**: 想定学習時間のページは作成しない

### まとめスライド

```markdown
---
<!-- _class: title-top -->

## [章番号] まとめ

<div class="no-bg">

**本章では以下の内容を学習しました。**

</div>

### [トピック1]

- ポイント1
- ポイント2

### [トピック2]

- ポイント1
- ポイント2

---
```

### 次章への誘導

**注意**: 「次章では〜」の文章は不要。まとめスライドで終了する。

---

## 8. 注意事項

### 避けるべきこと

- 1スライドに情報を詰め込みすぎない
- 長文の説明（簡潔な箇条書きに変換する）
- 空白スライドの作成（最後の `---` に注意）

### 確認すべきこと

- [ ] フッターが正しく表示されているか
- [ ] ページ番号が表示されているか
- [ ] 赤色のヘッダーバーが表示されているか
- [ ] 太字が赤色で表示されているか
- [ ] 2カラム/3カラムレイアウトが正しく表示されているか
- [ ] 画像が中央寄せされているか

- 詳細ポイント3
